Prompt: Hivio mobile wireframe/design direction
Design a mobile-first app concept for “Hivio,” a job application tracker made specifically for college students and recent graduates.

The app should not look like a generic AI-generated productivity app. Avoid the overly polished, copy-paste SaaS template feel. Instead, make it feel:

warm but professional
modern but not corporate
distinctive and memorable
student-centered
calm, organized, and trustworthy
visually clean without feeling empty or sterile
Core product purpose
Hivio helps students organize their job search in one place. Users can:

create an account and log in securely
set up a profile
upload and label resumes
add and manage job applications
link resumes to applications
track statuses like Applied, Interview, Rejected, Offer
add notes and reminders
review analytics about applications and resume performance
Platform
Design for a mobile app / mobile-first experience using an iPhone 14 frame (390 x 844 px).

Design direction
The UI should feel purpose-built, not template-based.

Use these design principles:

minimal clutter
strong spacing and hierarchy
rounded cards and soft structure
excellent readability and contrast
subtle personality
consistent components across screens
make status information visually easy to scan
avoid overly futuristic gradients, crypto-dashboard vibes, or generic startup illustrations
Brand personality
Hivio should feel like:

a smart companion during a stressful job search
supportive, focused, and organized
something a student would actually want to open daily
more thoughtful than LinkedIn, less boring than a spreadsheet
Visual style guidance
Create a unique visual identity with:

a clean modern mobile interface
soft but confident typography hierarchy
subtle branded accent color used intentionally
status colors that are meaningful and consistent
cards, chips, and sections that feel designed rather than default
elegant whitespace
light mode primary, with dark mode considered later
Avoid:

generic blue enterprise UI
random floating widgets
excessive shadows
cramped forms
“AI app” cliché visuals
sterile grayscale-only product styling in the final concept
Suggested brand feel
A good direction would be:

neutral base colors
one signature accent color that feels fresh and recognizable
one or two supporting colors
status badges for Applied / Interview / Rejected / Offer
soft surfaces, clear borders, and understated depth
Information architecture
Create wireframes or polished low/mid fidelity mobile screens for these pages:

1. Welcome / Login screen
Include:

Hivio wordmark or logo area
short supportive tagline
email field
password field
primary sign-in CTA
secondary CTA to create account
clean and calm layout
2. Registration screen
Include:

full name
email
password
confirm password
create account CTA
link back to login
friendly onboarding tone
3. First-time profile setup
Include:

profile photo placeholder
major / school / graduation year
career interests
preferred job types
location preference
save and continue CTA
4. Dashboard / Home screen
This should be the emotional center of the app. Include:

personalized greeting
quick summary cards:
total applications
interviews
follow-ups due
recent applications list
visually distinct status indicators
upcoming reminder section
quick add application button
bottom navigation
5. Add Job Application screen
Include:

company
position title
application date
status selector
linked resume selector
notes field
follow-up date
save CTA
6. Application Detail screen
Include:

company and role header
current status chip
timeline or update history
linked resume
notes section
reminder/follow-up info
edit / archive / delete actions
7. Resume Manager screen
Include:

uploaded resumes list
resume labels/tags
upload new resume CTA
simple sorting/filter idea
each resume should feel easy to scan
8. Analytics screen
Include:

total applications
status breakdown
performance by resume
clean charts/cards that are simple and student-friendly
make analytics understandable at a glance, not overly technical
9. Settings screen
Include:

profile access
theme toggle
notifications/reminders
sign out
simple and uncluttered controls
Navigation
Use a mobile bottom navigation bar with 4–5 destinations such as:

Home
Applications
Resumes
Analytics
Settings
Avoid top desktop-style navigation.

Layout rules
prioritize thumb-friendly spacing
primary actions should be easy to reach
use full-width or near-full-width mobile buttons
forms should be vertically stacked
cards should be easy to scan quickly
status should be understandable without reading too much
Status system
Make application statuses visually distinct and consistent:

Applied
Interview
Rejected
Offer
Use chips, pills, icons, borders, or color coding in a way that feels elegant and not childish.

Tone
The app should feel:

encouraging
focused
calming
competent
designed for real students dealing with uncertainty
Important constraint
Do not make this look like a generic AI-generated fintech dashboard, admin template, or startup boilerplate.
Do not use random filler visuals that distract from the workflow.
Make the design feel intentional, branded, and human.

Output goal
Create a cohesive set of mobile app screens for Hivio that are visually distinctive, student-friendly, and clearly support:

onboarding
application tracking
resume management
analytics
settings
The result should feel like a real product concept with personality, not a default component kit.
